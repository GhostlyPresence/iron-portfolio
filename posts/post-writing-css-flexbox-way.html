Writing CSS the Easy Way: FlexBox

Master the art of CSS layout by understanding hierarchy and the power of FlexBox — a guide to making responsive, logical designs.

If you have just started writing CSS code, it's highly likely that you are having a hard time making the page responsive or aligning components properly, or positioning components as you have in your mind. This article is about how to maintain the hierarchy of the page and, in turn, making CSS more logical.

The best way to learn is to open CodePen and write the code yourself. See what happens.

Maintaining the Hierarchy

Having a complete picture of how the page is structured is important, and <div> is your lifesaver. For example, if you are making a navigation bar, its structure will be something like this:

<div class="nav-bar">
  <div class="logo">
    <h1>LOGO</h1>
  </div>
  <div class="nav-links-container">
    <div class="nav-link">Home</div>
    <div class="nav-link">About</div>
    <div class="nav-link">Contact</div>
  </div>
</div>

This structure will help tremendously when you apply CSS to it.

Adding Background Color for Better Visualization

Here I have added different background colors to all the components. Now you can see clearly how much space each one is taking and how it is aligned with respect to its parent container.

Note: I have added padding so you can see more clearly, but you don't need to do that after you understand the structure.

.nav-bar {
  background-color: green;
  padding: 5px;
}

.logo {
  background-color: red;
}

.nav-links-container {
  background-color: blue;
  padding: 10px;
}

.nav-link {
  background-color: yellow;
  margin: 2px;
}

Using FlexBox

Now you know how to structure your nav bar, but it doesn't look quite right. Here comes FlexBox!

Flexbox is a module that aims at providing a more efficient way to lay out, align, and distribute space among items in a container. Set the display property of any container to flex, and all child components inside that container are arranged in sort of an array—either horizontally (default) or vertically.

.nav-bar {
  display: flex;
}
/* To change the direction to vertical, add:
   flex-direction: column; */

Sizing Child Elements with Flex Property

You can see the components are arranged horizontally, but the width doesn't look quite right. For this, we use the flex property.

Keep in mind: Only the child containers of the container having display set to flex can use the flex property.

The flex property defines the ratio of the total width/height the child containers will take. If we want the logo and nav-links-container to have equal width (1:1 ratio), we set flex: 1 in both:

.logo, .nav-links-container {
  flex: 1;
}
/* Change the value to 2:1 to see how it affects the layout */

Alignment in FlexBox

Now we can focus on the logo container and nav-links-container separately. In flexbox, there are two types of alignment:

  • Main-axis: The axis along which child components are aligned
  • Cross-axis: The perpendicular axis

Aligning Along the Main-Axis

To align child components along the main-axis, use justify-content: center:

.logo {
  justify-content: center;
}

Using Space-Around

To distribute nav-links more efficiently, use justify-content: space-around:

.nav-links-container {
  justify-content: space-around;
}

There are more values that can be given to justify-content that you can research on your own.

Aligning Along the Cross-Axis

To align nav-links along the vertical axis (cross-axis), use the align-items property:

.nav-link {
  display: flex;
  align-items: center;
}
}

Making It Responsive with Media Queries

The @media rule, introduced in CSS2, made it possible to define different style rules for different media types. This means you can have different CSS rules based on the screen size, viewport, or orientation.

For example, if the screen size decreases to 800px or less (indicating a mobile device), we can change the flex-direction to column:

@media (max-width: 800px) {
  .nav-bar {
    flex-direction: column;
  }
}

Key Takeaways

  • display: flex — Arrange components horizontally/vertically
  • flex-direction — Change the axis along which components are arranged
  • flex — Set the width of child components relative to the parent container
  • justify-content — Align components along the main-axis
  • align-items — Align components along the cross-axis
  • @media — Define different style rules for different media types

Before you go: Reading this for the first time, you might find it a bit tricky to understand. Try writing the code alongside as you read. Remember, the thinking process behind solving a problem is much more important than just being able to solve the problem.
